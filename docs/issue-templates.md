# 이슈·PR 템플릿

## 이슈 양식

| 순서 | 파일 | 용도 | 라벨 선택 |
|---:|---|---|---|
| 1 | `01-feature.yml` | 새로운 기능과 결과물 | 저장소 프로필 기준으로 수동 선택 |
| 2 | `02-ui-ux.yml` | 화면 구성과 사용성 개선 | 저장소 프로필 기준으로 수동 선택 |
| 3 | `03-content.yml` | 글, 이력과 소개 콘텐츠 | 저장소 프로필 기준으로 수동 선택 |
| 4 | `04-operations.yml` | 배포, 자동화와 환경 관리 | 저장소 프로필 기준으로 수동 선택 |
| 5 | `05-learning.yml` | 기술 조사와 실습 기록 | 저장소 프로필 기준으로 수동 선택 |
| 6 | `06-bug.yml` | 예상과 다른 동작과 오류 | 저장소 프로필 기준으로 수동 선택 |

파일은 [`.github/ISSUE_TEMPLATE`](../.github/ISSUE_TEMPLATE)에서 관리합니다. 전역 양식은 특정 라벨을 자동으로 붙이지 않습니다. 중요도, 난이도와 프로필별 라벨은 이슈 생성 후 선택합니다. 알고리즘 저장소의 `분류:` 라벨만 복수 선택할 수 있고, 나머지 그룹은 하나만 선택합니다.

## 상속 규칙

- 개인 계정의 공개 `.github` 저장소에 정의한 기본 템플릿은 다른 공개 저장소가 상속할 수 있습니다.
- 개별 저장소에 자체 `.github/ISSUE_TEMPLATE`이 있으면 해당 저장소 설정이 우선합니다.
- 중앙 템플릿 상속과 재사용 워크플로 실행은 별도 기능입니다.
- 비공개 저장소에는 개인 계정의 공개 기본 템플릿이 자동 적용되지 않습니다.

## Pull Request 템플릿

[PULL_REQUEST_TEMPLATE.md](../.github/PULL_REQUEST_TEMPLATE.md)는 다음 내용을 확인합니다.

- 변경 목적과 범위
- 구현 및 수정 내용
- 검증 결과
- 관련 이슈
- 배포와 호환성 영향

저장소 전용 검증 항목이 필요하면 해당 저장소에 자체 PR 템플릿을 둡니다.
