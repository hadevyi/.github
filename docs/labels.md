# 라벨 체계

라벨의 실제 생성·갱신 정의는 [reusable-sync-labels.yml](../.github/workflows/reusable-sync-labels.yml)에 있습니다. 이 문서는 사람이 확인하는 운영 명세입니다.

## 적용 방식

- 모든 관리 대상 저장소에는 `중요도:`와 `난이도:` 라벨만 공통으로 적용합니다.
- 저장소 성격에 맞는 프로필별 라벨을 추가합니다.
- `유형:` 라벨은 일반 프로젝트 프로필에서만 사용합니다.
- 알고리즘 저장소는 풀이 방법을 나타내는 `분류:` 라벨을 사용합니다.
- 동기화는 라벨을 생성하거나 이름·색상·설명을 갱신합니다.
- 중앙 정의에 없는 기존 라벨은 삭제합니다. 기존 이슈에 붙은 해당 라벨도 함께 제거됩니다.
- 이슈 양식은 공통으로 상속되므로 라벨을 자동 지정하지 않습니다. 이슈를 만든 뒤 저장소 프로필에 맞춰 선택합니다.

## 이슈 분할 원칙

- 이슈 하나는 독립적으로 검증하고 종료할 수 있는 결과 하나를 다룹니다.
- 기본적으로 `중요도:`와 `난이도:`는 각각 하나만 사용합니다.
- 프로필별 그룹도 기본적으로 하나만 선택합니다. 예를 들어 일반 프로젝트의 `유형:`과 `영역:`은 각각 하나씩 사용합니다.
- 알고리즘 저장소의 `분류:`는 한 문제에 여러 풀이 방식이 함께 쓰일 수 있으므로 복수 선택할 수 있습니다.
- 같은 그룹의 라벨이 둘 이상 필요하고 알고리즘 분류가 아니라면 이슈를 분리하고, 부수 범위와 관련 이슈는 본문에 적습니다.
- 여러 하위 작업을 추적하는 이슈만 예외로 두며, 제목이나 본문에 추적 목적을 명시합니다.

## 표시 순서

워크플로는 위 표의 라벨만 남기도록 생성·갱신·삭제를 수행합니다. GitHub에는 사용자 지정 라벨 정렬 순서를 저장하는 필드가 없으므로 Issues 화면과 라벨 선택기의 표시 순서는 보장되지 않습니다.

접두어를 통일해 이름 기준으로 표시될 때 같은 분류가 모이도록 합니다. 화면 순서를 강제하기 위한 숫자 접두어는 사용하지 않습니다.
