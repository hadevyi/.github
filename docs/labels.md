# 라벨 체계

라벨의 실제 생성·갱신 정의는 [reusable-sync-labels.yml](../.github/workflows/reusable-sync-labels.yml)에 있습니다. 이 문서는 사람이 확인하는 운영 명세입니다.

## 적용 방식

- 일반 관리 대상 저장소에는 공통 라벨을 적용합니다.
- 저장소 성격에 맞는 프로필별 라벨을 추가합니다.
- 중앙 설정 프로필은 `콘텐츠`, `학습`을 제외하고 중앙 전용 라벨을 추가합니다.
- 동기화는 라벨을 생성하거나 이름·색상·설명을 갱신합니다.
- 중앙 정의에 없는 기존 라벨은 삭제합니다. 기존 이슈에 붙은 해당 라벨도 함께 제거됩니다.
- 각 이슈에는 유형 1개, 중요도 1개와 난이도 1개를 기본으로 적용합니다.

|---|---|---|

| `project` | 백엔드, 프론트엔드, 데이터, 인프라, 인공지능 | `English-Word-Study`, `O-to-I`, 향후 일반 프로젝트 저장소 |

프로필은 각 호출 워크플로의 `profile` 입력으로 선택합니다.

## 표시 순서

워크플로는 위 표의 라벨만 남기도록 생성·갱신·삭제를 수행합니다. GitHub에는 사용자 지정 라벨 정렬 순서를 저장하는 필드가 없으므로 Issues 화면과 라벨 선택기의 표시 순서는 보장되지 않습니다.

접두어를 통일해 이름 기준으로 표시될 때 같은 분류가 모이도록 합니다. 화면 순서를 강제하기 위한 숫자 접두어는 사용하지 않습니다.

## 기존 학습 라벨 전환

| 기존 라벨 | 전환 라벨 |
|---|---|
| `Learn : Book` | `학습 자료: 도서` |
| `Learn : Official Doc` | `학습 자료: 공식 문서` |
| `Learn : Online Lecture` | `학습 자료: 온라인 강의` |
| `Level : Easy` | `난이도: 쉬움` |
| `Level : Nomal` | `난이도: 보통` |
| `Level : Hard` | `난이도: 어려움` |

`상태`는 GitHub Project의 `Todo`, `In Progress`, `Done`으로 관리하고 별도 라벨을 만들지 않습니다. 담당자는 이슈 양식과 자동 배정 워크플로로 관리합니다.
