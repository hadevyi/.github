# hadevyi 공통 GitHub 설정

`hadevyi` 계정의 공개 저장소에서 함께 사용하는 이슈·PR 템플릿과 재사용 워크플로를 관리합니다.

## 제공 항목

| 구분 | 항목 | 적용 방식 |
|---|---|---|
| Issue Form | 기능, UI/UX, 콘텐츠, 운영, 학습, 버그 | 개별 공개 저장소에 자체 템플릿이 없으면 자동 상속. 라벨은 저장소 프로필에 맞춰 수동 선택 |
| Pull Request | 공통 변경·검증 템플릿 | 개별 공개 저장소에 자체 템플릿이 없으면 자동 상속 |
| Actions | Astro Pages 빌드·배포 | 개별 저장소의 호출 워크플로에서 사용 |
| Actions | 신규 이슈 자동 배정 | 개별 저장소의 호출 워크플로에서 사용 |
| Actions | 공통 라벨 동기화 | 개별 저장소에서 월 1회 또는 수동 실행 |

기본 템플릿은 조건에 따라 자동 상속되지만, 재사용 워크플로는 각 저장소에 호출 파일이 있어야 실행됩니다.


## 적용 원칙

- 개별 저장소의 자체 이슈·PR 템플릿이 중앙 기본 템플릿보다 우선합니다.
- 저장소별 변수, Secret, 실행 이벤트와 권한은 호출 저장소에서 관리합니다.
- `.node-version`, `.vscode`, 빌드 스크립트와 저장소 전용 라벨은 중앙화하지 않습니다.
- 중앙 워크플로를 먼저 배포한 뒤 각 저장소의 호출 워크플로를 반영합니다.
- 아카이브, 삭제 예정, Fork 저장소에는 중앙 호출 워크플로를 추가하지 않습니다.
- 공개 저장소의 기본 템플릿 상속 여부와 호출 워크플로 적용 여부는 별개입니다.
